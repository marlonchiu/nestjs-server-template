# 质量门禁

## Gate 0：代码质量（开发完成后，测试执行前）

- [ ] TypeScript / 类型检查编译无错误
- [ ] Lint（ESLint / Biome）检查通过，无 error 级别问题
- [ ] 无已知高危依赖漏洞

## Gate 1：测试（QA 执行后，部署前）

阶段 3 完成后，必须全部通过才能进入阶段 4。

- [ ] 单元测试全部通过
- [ ] 测试覆盖率 ≥ 70%
- [ ] 无 P0/P1 级别 Bug
- [ ] E2E 测试已编写并通过
- [ ] 集成测试通过

## Gate 2：性能（部署前，仅适用于 Web 项目）

- [ ] 前端：Lighthouse Performance Score ≥ 80
- [ ] 后端：API P99 响应时间 < 500ms
- [ ] 无内存泄漏（长时间运行稳定）

## 判断标准

问自己：
- **如果现在把这个代码交给用户，他能正常使用吗？**
- **核心流程有没有遗漏的测试？**
- **有没有明显的性能或安全问题？**

## 未通过处理

如果门禁未通过：
1. 记录失败原因到 `qa-report.md`
2. 回到阶段 3 修复
3. 重新执行测试
4. 再次检查门禁

不允许绕过门禁直接部署。
